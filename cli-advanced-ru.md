# <a name="cli-advanced-usage"></a>Расширенное использование CLI

## <a name="cli-commands"></a>Команды CLI

### Основные команды

```bash
# Запустить CLI
spck

# Запустить мастер настройки
spck --setup

# Показать информацию об аккаунте
spck --account

# Выйти и удалить учётные данные
spck --logout

# Показать справку
spck --help

# Показать версию
spck --version
```

### Расширенные опции

```bash
# Использовать пользовательский файл конфигурации
spck --config /path/to/config.json
spck -c /path/to/config.json

# Переопределить корневой каталог
spck --root /path/to/project
spck -r /path/to/project

# Переопределить сервер ретрансляции (например, использовать конкретный регион)
spck --server cli-eu-1.spck.io
spck -s cli-na-1.spck.io
```

## <a name="ai-coding-agents"></a>ИИ-агенты для кодирования

Терминал Spck CLI предоставляет полный доступ к оболочке, что означает возможность запускать ИИ-агенты для кодирования прямо с мобильного устройства. Эти агенты могут читать, писать и рефакторить код в вашем проекте, пока вы наблюдаете из Spck Editor.

> 💡 **Совет**: Используйте **tmux**, чтобы сессии ИИ-агентов продолжали работать даже после отключения. Запустите сессию tmux на своём компьютере (`tmux new -s code`), запустите ИИ-агент, а затем повторно подключитесь из терминала Spck CLI на телефоне (`tmux attach -t code`). Это позволяет плавно переключаться между компьютером и телефоном без потери контекста. Полное руководство, включая настройку постоянного удалённого сервера, см. в [Использование Tmux](./tmux).

## <a name="advanced-usage"></a>Расширенное использование

### Несколько проектов

Запускайте отдельные экземпляры CLI для разных проектов одновременно:

```bash
# Терминал 1: Проект A
cd /path/to/projectA
spck

# Терминал 2: Проект B
cd /path/to/projectB
spck
```

Каждый проект сохраняет собственную конфигурацию и подключение.

> 💡 **Совет**: Вы также можете использовать несколько экземпляров CLI для передачи файлов между компьютером и телефоном. Пошаговое руководство см. в [Передача файлов между мобильным устройством и компьютером](./cli-file-transfer).

### Пользовательские файлы конфигурации

Создавайте специализированные конфигурации для разных сценариев:

```bash
# Конфигурация для разработки
spck --config ~/configs/dev-config.json

# Конфигурация для продакшена (только чтение, без терминала)
spck --config ~/configs/prod-config.json
```

### Настройка для конкретной среды

**Локальная разработка:**

```json
{
  "security": {
    "userAuthenticationEnabled": false
  },
  "terminal": {
    "enabled": true
  }
}
```

**Продакшен-сервер:**

```json
{
  "security": {
    "userAuthenticationEnabled": true
  },
  "terminal": {
    "enabled": false
  }
}
```

### Высокая нагрузка на CPU

Снизьте интенсивность отслеживания файлов, добавив больше паттернов игнорирования:

```json
{
  "filesystem": {
    "watchIgnorePatterns": [
      "**/.git/**",
      "**/.spck-editor/**",
      "**/node_modules/**",
      "**/dist/**",
      "**/build/**",
      "**/.next/**",
      "**/coverage/**",
      "**/.cache/**"
    ]
  }
}
```

Ограничьте количество одновременных терминалов:

```json
{
  "terminal": {
    "maxTerminals": 5
  }
}
```

## <a name="mobile-prompt"></a>Сокращение приглашения оболочки для мобильных устройств

На мобильных устройствах горизонтальное пространство экрана ограничено. Приглашение оболочки по умолчанию — обычно включающее путь текущего каталога, имя пользователя и имя хоста — может загромождать терминал и затруднять чтение вывода команд.

Изменение приглашения на просто `$ ` обеспечивает значительно более удобный терминальный опыт на маленьких экранах.

### Bash

Добавьте следующее в `~/.bashrc`:

```bash
export PS1='\$ '
```

Примените без перезапуска оболочки:

```bash
source ~/.bashrc
```

### Zsh

Добавьте следующее в `~/.zshrc`:

```zsh
PROMPT='$ '
```

Примените без перезапуска оболочки:

```zsh
source ~/.zshrc
```

### PowerShell (Windows)

Создайте файл профиля, если он ещё не существует, затем откройте его:

```powershell
New-Item -Path $PROFILE -Type File -Force
notepad $PROFILE
```

Добавьте следующее в профиль:

```powershell
function prompt { "$ " }
```

Примените без перезапуска оболочки:

```powershell
. $PROFILE
```

### Командная строка (Windows)

Установите минимальное приглашение для текущей сессии:

```cmd
PROMPT $$
```

Чтобы сделать это постоянным, добавьте `PROMPT` как переменную среды **пользователя** или **системы** со значением `$$` через **Панель управления → Система → Дополнительные параметры системы → Переменные среды**.
