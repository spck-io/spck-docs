# <a name="cli-language-server"></a>CLI Language Server

## <a name="cli-ls-overview"></a>Обзор

Spck CLI включает полноценный мост Language Server Protocol (LSP), который запускает языковые серверы для TypeScript, JavaScript, Python, HTML, CSS и SCSS непосредственно на вашем удалённом хосте. При подключении к сеансу CLI Spck Editor автоматически использует удалённый языковой сервер вместо встроенного мобильного.

> 💡 **Совет**: Описание встроенного языкового сервера, доступного без CLI, см. в разделе [Языковой сервер редактора](./editor-language-server).

## <a name="cli-ls-architecture"></a>Архитектура

Языковой сервер CLI работает на той же машине, что и файлы вашего проекта. Это даёт ряд существенных преимуществ по сравнению с подходом, работающим в браузере на мобильных устройствах:

### Отсутствие передачи файлов

При использовании встроенного мобильного языкового сервера файлы необходимо сначала передать на устройство и загрузить в браузерный воркер, прежде чем сервер сможет их проанализировать. С CLI языковой сервер читает файлы непосредственно с диска — передача не требуется. Это устраняет накладные расходы на пропускную способность и обеспечивает высокую скорость анализа даже в крупных проектах.

### Полное представление о проекте

Языковой сервер CLI имеет доступ ко всему каталогу вашего проекта, включая:

- Все исходные файлы в каждом подкаталоге
- `node_modules/` для разрешения типов сторонних библиотек
- Автоматически обнаруженные файлы `tsconfig.json` на любой глубине
- `.js`, `.ts`, `.d.ts`, `.jsx`, `.tsx`, `.vue` и связанные файлы объявлений

Это означает, что «Перейти к определению», «Найти ссылки» и «Переименовать символ» корректно работают через границы файлов — даже если вы не открывали эти файлы в редакторе.

### Правильная обработка `tsconfig.json`

Языковой сервер TypeScript автоматически обнаруживает и уважает `tsconfig.json` (или `jsconfig.json`) вашего проекта. Псевдонимы путей (`paths`), параметры компилятора (`strict`, `target`, `lib`), ссылки на проекты и структуры рабочего пространства монорепозитория — всё обрабатывается корректно. Встроенный мобильный языковой сервер не может читать `tsconfig.json`, поскольку при запуске у него нет доступа к файловой системе.

### Постоянный процесс

Процесс языкового сервера продолжает работать на протяжении всего сеанса CLI. Он строит индекс проекта в памяти и поддерживает его актуальным между правками, поэтому ответы остаются быстрыми даже после первоначального запуска.

## <a name="cli-ls-features"></a>Поддерживаемые функции

| Функция | Описание |
| --- | --- |
| **Автодополнение кода** | Контекстные предложения с сигнатурами типов, отсортированные по релевантности |
| **Информация при наведении** | Информация о типах и документация JSDoc для символа под курсором |
| **Помощь по сигнатуре** | Подсказки параметров и документация при вводе вызова функции |
| **Перейти к определению** | Перейти к объявлению любого символа, включая символы в `node_modules` |
| **Найти ссылки** | Список всех вхождений символа в проекте |
| **Переименовать символ** | Переименовать символ и атомарно обновить все ссылки |
| **Диагностика** | Подсветка ошибок и предупреждений в реальном времени по мере ввода |
| **Рендеринг Markdown** | Документация при наведении и сигнатуры, отображаемые как Markdown с блоками кода с подсветкой синтаксиса |

## <a name="cli-ls-config"></a>Конфигурация

Поддержка языкового сервера включена по умолчанию. Вы можете управлять ею в файле конфигурации:

```json
{
  "languageServer": {
    "enabled": true,
    "typescript": {
      "enabled": true
    },
    "python": {
      "enabled": true
    }
  }
}
```

- **`languageServer.enabled`** (boolean): Главный переключатель для всех удалённых языковых серверов. По умолчанию: `true`.
- **`languageServer.typescript.enabled`** (boolean): Включить языковой сервер TypeScript/JavaScript. По умолчанию: `true`.
- **`languageServer.python.enabled`** (boolean): Включить языковой сервер Python (требуется `pylsp` в `PATH`). По умолчанию: `true`.

## <a name="cli-ls-requirements"></a>Требования

- **TypeScript / JavaScript**: Встроен в CLI. Дополнительная установка не требуется.
- **Python**: Требуется [`python-lsp-server`](https://github.com/python-lsp/python-lsp-server) (`pip install python-lsp-server`).
- **HTML / CSS / SCSS**: Встроен в CLI. Дополнительная установка не требуется.

| Работа в офлайн-режиме (без CLI) | — | ✓ |
